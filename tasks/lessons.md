# Lessons Learned

- **2026-03-08**: Project initialized. Reference packet is Ashdown Classic Chevrolet 02/11/2026 (27 pages). All parsers must handle handwritten notes gracefully.
- **2026-03-08**: FloorplanReconciliation gets its own model — the variance between book value and floorplan balance is the most politically sensitive data point in the system.
- **2026-03-08**: R&R PDF formats vary between report types. Section identifiers need multiple variants. Parser must be defensive — log warnings on unparseable rows, never crash.
- **2026-03-08**: F&I chargeback section identifiers (850, 851) are too short for simple substring matching — need context-aware `can_handle` to avoid false positives.
- **2026-03-08**: For 'lt' comparisons (like parts turnover), yellow threshold must be HIGHER than red — yellow at <2.0 means 'getting low,' red at <1.0 means 'critical.' Check red first to avoid double-flagging.
- **2026-03-08**: ReportLab's Table class handles page breaks for long tables automatically with `repeatRows` parameter — use it so column headers repeat on continuation pages. Alternating row colors via TableStyle with `ROWBACKGROUNDS`.
- **2026-03-08**: When testing PDF generators, bypass the async DB layer by calling section builder methods directly with mock data dicts. Use pdfplumber for round-trip validation (generate → extract text → assert content).
- **2026-03-08**: FastAPI's UploadFile with Form() fields works for multipart uploads. Use Body() for JSON payloads in non-upload routes. Pydantic response schemas with response_model give you auto-validation and OpenAPI docs for free.
- **2026-03-08**: Python 3.9 doesn't support `str | None` or `list[X]` in runtime annotations. Use `from __future__ import annotations` or `Optional[str]` / `List[X]` from typing. Always check target Python version.
- **2026-03-08**: For httpx test client with FastAPI bulk file uploads, the `files` param field name must match the FastAPI parameter name exactly (e.g., `files` not `file`).
- **2026-03-09**: For Phase 1, a single HTML file with vanilla JS served by FastAPI is faster than setting up Next.js. Save the full frontend framework for Phase 2 when we need the accountability dashboard, auth, and real-time updates.
- **2026-03-09**: Scanned PDFs (no text layer) require OCR. Use easyocr + pypdfium2 (pure Python, no system deps like tesseract/poppler). Lazy-load the OCR reader to avoid startup cost. Mark pages with `ocr_used=True` so parsers can adapt.
- **2026-03-09**: OCR table reconstruction: group bounding boxes by y-coordinate (tolerance ~15px), sort within row by x-coordinate. This gives synthetic table rows without needing pdfplumber's table detection.
- **2026-03-09**: OCR number artifacts: `~` means negative, `$` at end means `%`, double commas (`1,,688.80`), trailing dots, spaces inside numbers (`0 . 0 1`). Handle each specifically — never blindly clean entire lines or you'll merge separate column values.
- **2026-03-09**: For OCR text with spaced numbers, do NOT pre-clean entire lines with `_clean_ocr_line()` — it collapses separate column values into garbage (e.g. `274 100 691 29941.28` → `27410069129941.28`). Instead: 1) match regex on raw lines, 2) clean only the captured value, 3) second pass with cleaned lines but skip TOTAL lines.
- **2026-03-09**: OCR misspells common words: `BALANCE` → `BALLCE`, `BALWICE`, `BALACE`. Use `BAL\w*` in regex patterns instead of `BALANCE`. Same for `ACCOUNT` → `ACCOU:T`, `AccOUNT`.
- **2026-03-09**: Continuation pages (page 2 of N) don't have section identifiers. Detect them via column headers containing schedule numbers (237, 240, 277) + "SCHEDULE SUMMARY" + "DESCRIPTION".
- **2026-03-09**: Slow-to-accounting dates in OCR show `'DD/YY` format where the month prefix was dropped. If second number is 20-40, treat as DD/YY (day, year) with meeting month implied.
- **2026-03-09**: OCR column bleed: when columns are close together, OCR merges digits across column boundaries (e.g. `443,143.27` should be `43,143.27`). Heuristic fix: if value > 2x expected magnitude, try stripping leading digit.
- **2026-03-09**: OCR adds lowercase letter suffixes (especially 'v') to stock numbers from checkmarks/annotations. Always use `re.IGNORECASE` on OCR line patterns, and clean trailing `v/V/+` from captured stock numbers.
- **2026-03-09**: OCR replaces digits with similar characters: `+` for digits (7704+0 = 77040), `[` for 1, space inside numbers (7716 1 = 77161), `:` for digits. RO/claim number regexes must tolerate these.
- **2026-03-09**: Never include data column values (like "CWI") in header-skip regexes. `CWI` appears both in column headers AND as a data field in RO lines. Match header patterns more specifically (e.g. "Advisor Dept" not just "Dept").
- **2026-03-09**: For OCR cost_of_sales extraction, TOTAL prefix must be mandatory — the header line "coST SALES PARTS PIECES KTD MTD YTD" combined with the first data row creates a false match when TOTAL is optional.
- **2026-03-09**: OCR uses `"` and `'` as negative sign prefixes (e.g. `"24,900.00` = `-24,900.00`). Replace these with `-` before currency parsing.
- **2026-03-09**: When testing PDF generators without a DB, use MockObj with `__getattr__` returning None for missing attrs, and wrap string enum fields in _EnumLike objects that provide `.value` and `__hash__`.
- **2026-03-09**: FlaggingEngine._evaluate_record works with any object that supports getattr — just wrap parsed dicts in a simple class. No DB needed for flag evaluation.
- **2026-03-09**: Executive summary should be computed from actual parsed data, not stored separately — keeps it always in sync. Use efficient aggregation queries (COUNT, SUM with filters) rather than loading all records.
- **2026-03-09**: When adding new SQLAlchemy models with relationships to existing models, ensure the string-referenced class name in `Mapped[List["ClassName"]]` matches the actual class name exactly (e.g., `FlagResponseRecord` not `FlagResponse`).
- **2026-03-09**: 10-character minimum on flag responses prevents cop-out answers. Corporate wanted accountability, not checkboxes.
- **2026-03-09**: Recurring flag detection compares by category + field_name + field_value (stock_number, ro_number, etc.) — not just by message text, which can change slightly between meetings.
- **2026-03-09**: SendGrid deliverability requires SPF, DKIM, and DMARC DNS records. Also need custom FROM domain — sending from a generic domain gets flagged. - **2026-03-09**: Email service must be gracefully disabled when API key isn't configured — this is critical for local development and testing. Never let email failures crash the processing pipeline.
- **2026-03-09**: APScheduler with AsyncIOScheduler integrates cleanly with FastAPI — runs in the same event loop. Use US/Central timezone for all scheduled jobs.
- **2026-03-09**: When adding auth to existing routes, default test fixtures to corporate role for backward compatibility. Add role-specific tests separately in a dedicated RBAC test file. Never break green tests.
- **2026-03-09**: Store-scoped access control needs a UserStore join table — can't rely on role alone when GMs/managers may manage multiple stores. Corporate bypasses all store checks. Use `get_user_store_ids()` for list endpoints and `_user_has_store_access()` for single-resource endpoints.

- **2026-03-09**: Multi-stage Docker build for Next.js with output: 'standalone' reduces image size from ~1GB to ~150MB.
- **2026-03-10**: When deploying to a server that already has a reverse proxy (Traefik), don't bundle your own (Caddy). Use Docker labels for Traefik routing instead. Use priority values to ensure specific path prefixes (like /api) match before catch-all routes. Services that need external routing must join both the Traefik network ("web") and the internal network.
- **2026-03-10**: On shared servers, follow existing conventions (e.g., /opt/{project}/ for app directories, external "web" Docker network for Traefik). Setup scripts should be idempotent — check for existing Docker, users, networks, swap before creating.
- **2026-03-10**: With no host port bindings (Traefik handles ingress), health checks must use `docker compose exec` to curl from inside the container, not localhost from the host.
- **2026-03-10**: Traefik PathPrefix routing must be specific — use `/api/v1` not `/api` when NextAuth uses `/api/auth/*` paths. The broader prefix intercepts NextAuth callbacks/signin/error routes and sends them to FastAPI, which returns 404.
- **2026-03-10**: Frontend Dockerfiles should handle missing `package-lock.json` gracefully (e.g., `COPY package-lock.json* ./` with glob) since not all projects use npm lockfiles.
- **2026-03-10**: Alembic `env.py` must read `DATABASE_URL` from environment variables, not hardcode `localhost`. In Docker, the database hostname is the service name (e.g., `db`), not `localhost`.
- **2026-03-10**: Always add `tzdata` to Docker images that use timezone-aware scheduling (e.g., APScheduler with `US/Central`). Slim/Alpine images don't include timezone data by default.
- **2026-03-10**: Leverage existing infrastructure rather than adding parallel services. If the server already runs Traefik, use Traefik labels — don't add Caddy alongside it. Reduces complexity, avoids port conflicts, and uses proven config.
- **2026-03-11**: The stores table has a NOT NULL constraint on `code` — always check table schema before INSERT. Don't assume nullable columns.
- **2026-03-11**: Most dealership asset meeting packets are scanned images with no extractable text layer. pdfplumber returns empty strings — OCR is required. tesseract + pytesseract is lightweight and fast for batch extraction vs EasyOCR which needs GPU or heavy CPU.
- **2026-03-11**: Reynolds store numbers appear in parts analysis headers as "STORE XX BRANCH 01 {DEALER NAME} MM/YY" and in schedule footers as "Store: XX - {DEALER NAME}". These are the authoritative Reynolds site identifiers for mapping packets to stores.
- **2026-03-11**: Reynolds uses different legal entity names than the dealership brand names (e.g., "Classic Auto Park" for CAP, "Orr Motors of Arkansas" for Hot Springs Cadillac, "Orr Motors North" for Searcy CDJ). The store number is the reliable identifier, not the name.
- **2026-03-11**: Some packets (like Searcy CDJ meeting recap, CAP Receivables) are manually created cover sheets without Reynolds parts analysis pages — they won't have a STORE XX header. Need to identify these by content (e.g., cover sheet dealership name) or filename.
- **2026-03-11**: When SSH'ing for long-running operations (like OCR on many files), connections can timeout/reset. Use `timeout` command inside `docker exec` and keep operations focused (2 pages per file, not all pages).
- **2026-03-11**: Reynolds 7-digit site IDs (e.g., 7685787) do NOT appear in printed R&R reports/schedules via OCR. The 2-digit store numbers (STORE XX BRANCH 01) are the reliable identifiers in PDFs. The 7-digit IDs are system-level identifiers stored in the database only.
- **2026-03-11**: For packet completeness validation, keyword detection rates vary wildly across OCR'd scanned documents. Parts 2222 ("2222") hits 94% but Service/Parts Receivables and GL 0504 New/Used hit 0% — these reports use different header formats in scanned output. Need to refine detection patterns against actual report samples.
- **2026-03-11**: When running Alembic migrations in Docker, files on the host filesystem (/opt/...) are NOT automatically visible inside the container. Must `docker cp` new migration files into the running container OR rebuild the image. The container uses its own filesystem from the build image.
- **2026-03-11**: S3 services should lazy-load the boto3 client (don't import at module level) to avoid ImportError when boto3 isn't installed locally. Same pattern as EasyOCR lazy loading — expensive/optional imports go in the method that needs them.
- **2026-03-11**: For document type disambiguation in PDFs, use a scoring system: count matching patterns per document type, add priority weights for specificity, and use negative patterns to exclude overlapping matches (e.g., "0504" + "CHARGEBACK" = GL 0504 Chargebacks, not GL 0504 New/Used).
- **2026-03-11**: Split upload into validate-then-approve for user review. Upload saves file + runs PacketValidator only (meeting stays PENDING). Separate approve endpoint triggers full pipeline. This prevents processing unwanted/incomplete packets.
- **2026-03-11**: When refactoring endpoints that tests mock via `patch()`, keep imports at module level (not inside functions) so `patch("app.api.routes.upload.PacketValidator")` resolves correctly. Local `from X import Y` inside functions makes the mock target ambiguous.
- **2026-03-11**: Use sessionStorage to pass validation data between upload and review pages in Next.js — avoids an extra API round-trip and keeps the flow snappy. Key by meeting_id for uniqueness.
- **2026-03-11**: PacketValidator must have OCR fallback for scanned PDFs — pdfplumber returns blank text for image-only pages. Reuse the existing `_ocr_page_detailed` and `_build_text_and_tables` from `pdf_extractor.py` rather than duplicating OCR logic. Threshold: if pdfplumber text < 50 chars, try OCR.
- **2026-03-11**: OCR produces different text patterns than pdfplumber. Schedule numbers appear as "Schedule#: NNN" (not "SCHEDULE NNN" or "(NNN)"), warranty shows "WARR CLAIMS" (not "WARRANTY CLAIMS"), new vehicle inventory shows "NEW VEH" (not "NEW VEHICLE"). Keyword patterns must be OCR-tolerant with flexible regex.
- **2026-03-11**: easyocr and pypdfium2 must be in requirements.txt for Docker — they were lazily imported but never installed in the container. Also use CPU-only PyTorch (`--index-url https://download.pytorch.org/whl/cpu`) to keep Docker image ~1.5GB smaller.
- **2026-03-11**: CIT is Schedule 205, not Schedule 200. Schedule 200 is Accounts Receivable (P&S). Confusing these breaks both document type detections.
- **2026-03-11**: For long-running validation (OCR on 30+ page scanned PDFs), make the endpoint async: save file, return immediately, run validation in background thread via `asyncio.get_event_loop().run_in_executor()`. Frontend polls a progress endpoint. In-memory dict is fine for single-process apps — no Redis needed.
- **2026-03-11**: When mocking `asyncio` in tests (for `run_in_executor`), patch the entire `asyncio` module at the route level (`patch("app.api.routes.upload.asyncio")`), not individual functions. This prevents the background task from actually running during tests.
- **2026-03-12**: When a background task updates an in-memory progress store, `--workers N` (N > 1) with uvicorn multiprocessing breaks it — each worker has its own memory space. Progress set in worker A is invisible to worker B. Use `--workers 1` for apps with in-memory shared state, or use Redis/file-based stores for multi-worker.
- **2026-03-12**: For long-running page-by-page operations (like OCR + classification), do NOT extract all pages upfront then iterate — the progress stays at 0 during the entire extraction phase. Instead, extract and process one page at a time inside the progress loop so each page's completion updates the progress store immediately.
- **2026-03-12**: Always keep Dockerfile dependencies in sync with actual code. After switching from EasyOCR to tesseract, easyocr and torch were still in requirements.txt/Dockerfile — wasting 180MB+ of container memory and minutes of build time for unused packages.
- **2026-03-12**: When adding new columns to SQLAlchemy models, always create the corresponding Alembic migration immediately — not just the model change. Missing migrations cause `UndefinedColumnError` at runtime, which can crash background jobs (like schedulers) silently without user-visible errors.
- **2026-03-13**: Schedule number extraction (`Schedule#: NNN`) is far more reliable than keyword matching for classifying Schedule Summaries. Keywords like "ACCOUNTS RECEIVABLE" or "WARRANTY" can appear in multiple document types, but schedule numbers (200, 205, 220, 237, 240, 263, 277) are unique identifiers. Always try schedule number first, fall back to keywords only when OCR garbles the number.
- **2026-03-13**: For GL 0504 pages, the account number (15A, 15B, 850, 850A, 851, 851A) is the primary differentiator, not keywords like "CHARGEBACK" or "POLICY ADJ". Keywords are shared across subtypes but account numbers are unique. Extract account number first; only use keyword fallback when OCR garbles the "ACCOUNT" prefix.
- **2026-03-13**: Continuation pages (page 2+ of multi-page documents) need previous-page context to classify correctly. Schedule Summary continuation pages say "Schedule Summary" but have NO schedule number line. Without tracking `prev_doc_id`, these pages end up unclassified. Always pass previous page's classification to the next page's classifier.
- **2026-03-13**: Tesseract OCR from scanned PDFs produces predictable, consistent artifacts that can be accounted for with regex tolerance: "S"→"5" in "Schedule", truncated line starts ("hedule" for "Schedule"), "O"→"D"/"0" in "Open", garbled "ACCOUNT"→"ACCCUNT"/"A00010;T", lowercase mixed into uppercase ("TtTLE" for "TITLE"). Build regex patterns from actual OCR output samples, not from what the text "should" say. A reference OCR corpus (raw tesseract output per page) is invaluable for building robust patterns.
- **2026-03-13**: Python 3.9 doesn't support `float | None` syntax in class bodies — use `Optional[float]` from typing or `from __future__ import annotations`. The `__future__` import only helps with string annotations; SQLAlchemy's `Mapped[]` needs actual runtime types, so use `Optional[]`.
- **2026-03-13**: When overriding dataclass fields (like FlagRule thresholds), use `dataclasses.replace()` to create a new instance instead of mutating the original. This prevents side effects where one store's overrides corrupt the defaults for all subsequent evaluations.
- **2026-03-13**: Never hard-redirect (`window.location.href`) on 401 in a NextAuth app — NextAuth session survives the redirect and the authenticated-user redirect fires, creating an infinite loop. Instead, call `signOut()` from `next-auth/react` to clear the session first, which breaks the cycle. Also guard API fetches behind a token check so requests don't fire while the session is still loading.
- **2026-03-13**: PostgreSQL enum types require `ALTER TYPE ... ADD VALUE` to add new values — this cannot run inside a transaction. Use `op.execute()` in Alembic migrations with `IF NOT EXISTS` for idempotency. SQLite (used in tests) ignores this since enums are just strings there.
